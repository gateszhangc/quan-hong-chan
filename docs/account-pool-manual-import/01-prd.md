## 2. 背景与目标

### 2.1 背景
EasyClaw 平台采用**管理员预置账号池**模式，需要一种简单可靠的方式让管理员将 OpenAI ChatGPT 账号导入系统。

由于 OpenAI OAuth 回调域名限制，无法实现全自动授权流程，因此采用**手动复制粘贴**方案：管理员在本地完成 OAuth 授权，然后将生成的 Token 文件内容粘贴到平台导入。

### 2.2 目标
- 提供清晰的四步引导流程，降低管理员操作门槛
- 支持解析 OpenClaw/Codex CLI 生成的 `auth-profiles.json` 格式
- 加密存储 Token 信息，确保安全性
- 提供账号列表管理功能（查看、删除、状态监控）

### 2.3 用户
- **主要用户**：平台管理员（需要有管理权限）

## 3. 用户故事

### US-001: 管理员导入新账号
> 作为管理员，我希望通过简单的四步操作将 ChatGPT 账号导入平台，以便为用户分配使用。

**验收标准：**
- 清晰的步骤指引（安装 → 授权 → 复制 → 粘贴）
- 支持 Mac/Linux/Windows 不同系统的命令
- 自动解析 JSON 并验证格式
- 重复账号检测和提示

### US-002: 管理员查看账号列表
> 作为管理员，我希望查看所有账号的状态（总数量、已绑定、可用、失效），以便了解账号池健康状况。

**验收标准：**
- 顶部统计卡片展示四个核心指标
- 列表展示账号详情（ID、状态、绑定用户、时间）
- 支持删除未绑定的账号

### US-003: 管理员删除无效账号
> 作为管理员，我希望删除已失效或未使用的账号，以便保持账号池整洁。

**验收标准：**
- 未绑定账号可删除
- 已绑定账号禁止删除（有用户在使用）
- 删除前二次确认

### US-004: 管理员解绑账号
> 作为管理员，我希望将用户与账号解绑，以便回收账号分配给其他用户或处理异常情况。

**验收标准：**
- 已绑定账号显示"解绑"按钮
- 解绑前显示确认弹窗，告知影响（停止部署等）
- 解绑后自动停止该用户的所有部署
- 账号状态变为"可用"
- 保留解绑历史记录用于审计

### 5.2 添加账号页 (`/admin/accounts/add`)

四步向导界面：
1. **安装 OpenClaw** - 展示安装命令，一键复制
2. **登录授权** - 展示 onboard 命令，操作说明
3. **获取 Token** - 根据不同系统展示对应命令
4. **粘贴导入** - 文本框粘贴 JSON，显示解析结果，导入按钮

## 7. 非功能需求

### 7.1 性能
- 页面加载 < 2s
- 导入操作 < 3s（包含可选的 Token 验证）
- 列表页支持分页（前期账号少时可不分页）

### 7.2 可用性
- 操作步骤清晰，有复制按钮减少手动输入错误
- 错误提示具体（如"JSON 格式无效，请检查是否复制完整"）
- 导入成功后有明确的跳转提示

## 9. 附录

### 9.1 术语表
| 术语               | 说明                                            |
| ------------------ | ----------------------------------------------- |
| auth-profiles.json | OpenClaw 存储 OAuth Token 的文件                |
| account_id         | OpenAI 账号唯一标识（通常是邮箱或用户 ID）      |
| access_token       | OAuth 短期访问令牌（1小时有效）                 |
| refresh_token      | OAuth 长期刷新令牌（用于获取新的 access_token） |
