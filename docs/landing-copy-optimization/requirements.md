# EasyClaw 首页文案优化需求文档

## 项目背景

EasyClaw 是一个帮助用户快速部署 OpenClaw AI 助手的平台。当前首页部分模块文案需要优化：
1. Usage 模块步骤描述可以强化卖点
2. Feature 模块需要突出"零维护"和"Before/After对比"
3. FAQ 需要移除"免费试用"相关文案

## 优化目标

1. **强化核心卖点** - 突出"60秒部署"和"零维护"两大核心价值
2. **优化步骤描述** - Usage 模块增加模型灵活性和多渠道管理描述
3. **融入对比内容** - Feature 模块加入 Before/After 对比价值
4. **统一语言风格** - 各语言页面使用对应语言文案，不混搭

## 核心卖点

### 卖点 1：极速部署
- 60 秒内完成 OpenClaw 部署
- 从点击到运行，只需一分钟

### 卖点 2：零维护托管
- 平台自动处理服务器维护
- 自动更新 OpenClaw 版本
- 7×24 小时监控和故障自愈
- 用户只需专注使用 AI 助手

### 卖点 3：模型灵活性
- 支持 Claude、GPT、Gemini 多种模型
- 不同任务切换最适模型

### 卖点 4：多渠道统一管理
- 支持 Telegram、Discord、WhatsApp
- 一个后台统一管理所有渠道

## 文案风格要求

### 语调定位
- **正式程度**：专业但友好（Professional but friendly）
- **品牌个性**：简洁、高效、可靠、用户至上
- **人称**：多用"你"，少用"用户"
- **句子长度**：短句为主，最长不超过 20 字
- **情感**：积极、赋能、减少焦虑

### 禁止词汇
```
❌ streamline, optimize, innovative, cutting-edge, seamless, robust
❌ 赋能、生态、闭环、抓手、落地、颗粒度
❌ 免费试用（根据业务要求移除）
```

## 多语言文案要求

### 原则
- **英文页面** (`en.json`)：全部使用英文文案
- **中文页面** (`zh.json`)：全部使用中文文案
- **不混搭**：不在英文配置中使用中文，反之亦然

### 同步要求
- 英文页面修改后，中文页面需同步对应修改
- 保持语义一致，表达方式符合各自语言习惯
