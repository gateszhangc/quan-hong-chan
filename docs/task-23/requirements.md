# 任务23 - 需求文档

## 背景
当前 EasyClaw 首页导航栏缺少明显的 Blog 和 Pricing 入口按钮。用户希望通过按钮形式在导航栏右侧展示这两个重要页面的入口，以提升用户导航体验。

## 目标
在首页导航栏右侧添加 **Blog** 和 **Pricing** 按钮，点击后分别跳转到对应页面。

## 功能需求

### FR1: Blog 按钮
- **位置**: 导航栏右侧按钮组
- **样式**: Outline 边框按钮（次要样式）
- **图标**: RiArticleLine
- **点击行为**: 跳转到 `/posts` 页面
- **悬停效果**: 显示下划线或背景色变化

### FR2: Pricing 按钮
- **位置**: 导航栏右侧按钮组，位于 Blog 按钮右侧
- **样式**: Default 实心按钮（主要样式，突出显示）
- **图标**: RiMoneyDollarCircleLine
- **点击行为**: 跳转到 `/pricing` 页面
- **埋点追踪**: 自动触发 `pricing_viewed` 事件
- **悬停效果**: 背景色加深

### FR3: 响应式适配
- **桌面端**: 按钮完整显示，带有图标和文字
- **移动端**: 按钮移至汉堡菜单底部，保持相同功能

### FR4: 多语言支持
- 按钮文案需支持 i18n 国际化
- 英文: "Blog", "Pricing"
- 中文: "博客", "价格"

## 非功能需求

### NFR1: 性能
- 按钮渲染不应影响首屏加载时间
- 埋点追踪异步执行，不阻塞跳转

### NFR2: 可访问性
- 按钮需支持键盘导航（Tab 键）
- 按钮需有明确的 aria-label

### NFR3: 兼容性
- 支持 Chrome、Firefox、Safari、Edge 最新两个版本

## 验收标准

| 编号 | 验收项 | 优先级 |
|------|--------|--------|
| AC1 | 桌面端导航栏右侧显示 Blog 和 Pricing 按钮 | P0 |
| AC2 | Blog 按钮点击后正确跳转到 /posts | P0 |
| AC3 | Pricing 按钮点击后正确跳转到 /pricing | P0 |
| AC4 | Pricing 按钮点击触发 pricing_viewed 埋点 | P1 |
| AC5 | 移动端按钮在汉堡菜单中正常显示 | P1 |
| AC6 | 多语言切换时按钮文案正确更新 | P1 |
