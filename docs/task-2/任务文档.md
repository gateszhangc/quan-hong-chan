# 任务文档（基于详细设计文档）

## 任务目标
- 完成“登录 → 输入 Telegram Bot Token → 一键部署 → 返回结果”的最小可用流程。
- 模型固定 GPT-5.2，仅支持 Telegram。

## 任务拆分

1. 环境配置
- 定义并记录必要环境变量：`OPENAI_API_KEY`、`OPENAI_MODEL=gpt-5.2`、`DATABASE_URL`、`ENCRYPTION_KEY`、`DOCKER_HOST`。
- 确认部署环境可访问 Docker Socket。

2. 数据模型初始化
- 建表 `users`（`id`, `email`, `created_at`）。
- 建表 `deployments`（`id`, `user_id`, `status`, `telegram_token_encrypted`, `error_message`, `created_at`, `updated_at`）。

3. 登录模块
- 实现 Email 登录或 Google OAuth（任选其一）。
- 登录后进入部署页。

4. 首页实现（按 SimpleClaw 版式）
- 前端页面与组件必须使用 `frontend-design`（front-design）skill。
- 网站整体视觉风格与 https://shipany.ai/ 保持一致（色彩、字体、布局节奏、动效风格）。
- 顶部导航：`SimpleClaw.com` + `Contact Support` 文案。
- Hero：主标题 + 副标题（与版式一致）。
- 模型选择区：三张卡片（Claude Opus 4.5 / GPT-5.2 / Gemini 3 Flash），其中仅 GPT-5.2 可选，其他为禁用态。
- 通道选择区：Telegram 可选；Discord 禁用；WhatsApp 标注 `Coming soon`。
- 登录按钮：`Sign in with Google`，下方显示 `Checking availability...`。
- 对比区：Traditional vs SimpleClaw 的双列卡片结构。
- 用例区：多行短语块 + PS 提示。
- 页脚：`Built with love by Savio Martin · Contact Support`。
- 交互：模型/通道卡片可选态与禁用态；未登录点击触发 OAuth。

5. 部署接口（创建）
- 实现 `POST /api/deploy`。
- 校验 Telegram Token 不能为空。
- 加密 Token 并写入 `deployments`，状态置为 `provisioning`。
- 调用部署服务启动容器。

6. 容器部署服务
- 使用 Dockerode 创建容器并启动。
- 镜像：`openclaw/core:latest`。
- 环境变量注入：`TELEGRAM_TOKEN`、`OPENAI_API_KEY`、`OPENAI_MODEL=gpt-5.2`。
- 启动成功：更新 `deployments.status=running`。
- 启动失败：更新 `deployments.status=failed` 并写入 `error_message`。

7. 部署接口（查询）
- 实现 `GET /api/deploy/:id`。
- 返回 `status` 与 `error_message`。

8. 前端页面
- 登录页。
- 部署页：输入 Telegram Bot Token + 一键部署按钮。
- 结果页：展示部署成功/失败。

9. 前端状态轮询
- 部署成功后启动轮询 `GET /api/deploy/:id`。
- 直到返回 `running` 或 `failed`，停止轮询并更新 UI。

## 完成标准
- 登录后可提交 Telegram Token。
- 触发部署并在 60 秒内返回成功/失败。
- 前端正确显示部署结果。
