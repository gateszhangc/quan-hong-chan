# 后端详细设计文档（MVP）

## 1. 架构概览
```
Vercel (Frontend)
  -> /api/* 代理
     -> VPS (Backend API :5000)
        -> Docker Engine
           -> OpenClaw Container
        -> Postgres/Supabase
```

## 2. 技术栈
- Node.js
- Dockerode（控制 Docker 容器）
- Postgres / Supabase（持久化部署记录）
- AES-256（Telegram Token 加密）

## 3. 接口设计

### 3.1 创建部署
`POST /api/deploy`
- Header：`Authorization: Bearer <token>`
- Body：`{ "telegram_token": "..." }`
- Response：`{ "deployment_id": "uuid", "status": "provisioning" }`

### 3.2 查询部署
`GET /api/deploy/:id`
- Header：`Authorization: Bearer <token>`
- Response：`{ "status": "running" | "failed", "error_message"?: "..." }`

## 4. 数据模型

### 4.1 users
- `id` TEXT（来自认证 token 的用户标识）
- `email` TEXT（可选）
- `created_at` TIMESTAMP

### 4.2 deployments
- `id` UUID
- `user_id` TEXT
- `status` ENUM(`provisioning`, `running`, `failed`)
- `telegram_token_encrypted` TEXT
- `error_message` TEXT
- `created_at` TIMESTAMP
- `updated_at` TIMESTAMP

### 4.3 建表 SQL（Postgres）
```sql
-- users
create table if not exists users (
  id text primary key,
  email text unique,
  created_at timestamptz default now()
);

-- deployments
create table if not exists deployments (
  id uuid primary key,
  user_id text not null,
  status text not null check (status in ('provisioning','running','failed')),
  telegram_token_encrypted text not null,
  error_message text,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

create index if not exists idx_deployments_user_id on deployments(user_id);
create index if not exists idx_deployments_status on deployments(status);
```


## 6. 容器部署流程
1. 校验 Telegram Token。
2. 生成部署记录（`provisioning`）。
3. 使用 Dockerode 创建容器。
4. 注入环境变量并启动。
5. 成功：更新 `running`。
6. 失败：更新 `failed` 并记录错误。

## 7. 容器配置
- 镜像：`openclaw/core:latest`
- 启动命令：`npm start`
- 环境变量：
  - `TELEGRAM_TOKEN`（解密后注入）
  - `OPENAI_API_KEY`（服务端环境变量）
  - `OPENAI_MODEL=gpt-5.2`
- 端口：不暴露入站端口

## 8. 安全与加密
- `telegram_token` 入库前 AES-256 加密。
- 加密密钥通过环境变量 `ENCRYPTION_KEY` 提供。
- 仅在容器启动时解密使用。

## 9. 运行与部署
- 后端容器通过 Docker 部署在 VPS。
- 服务监听 `0.0.0.0:5000`。
- 前端通过 Vercel rewrite 代理访问：
  - `http://47.253.91.60:5000`

## 10. 环境变量
- `PORT=5000`
- `OPENAI_API_KEY=...`
- `OPENAI_MODEL=gpt-5.2`
- `DATABASE_URL=...`
- `ENCRYPTION_KEY=...`
- `DOCKER_HOST=unix:///var/run/docker.sock`

## 12. 健康检查
- 提供 `GET /health`，返回 `{ "ok": true }`。
- 部署后用于验证 `http://47.253.91.60:5000/health` 是否可访问。

## 13. Docker 部署步骤（明确命令）
1. VPS 上拉取代码：
```
git clone <repo_url>
cd backend
```
2. 启动服务：
```
docker compose up -d --build
```
3. 验证：
```
curl http://47.253.91.60:5000/health
```
