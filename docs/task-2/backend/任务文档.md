# 后端任务文档（MVP）

## 任务目标
- 在 VPS 上通过 Docker 部署后端服务。
- 提供部署创建与状态查询接口。
- 固定 GPT-5.2 + Telegram。

## 任务拆分

1. 初始化后端工程
- 在仓库根目录创建 `backend/`。
- 初始化 Node.js 项目与依赖（Express/Fastify、Dockerode、数据库客户端、加密库）。
- 目录结构需与详细设计文档 `## 11. 后端目录结构` 保持一致。

2. 环境变量与配置
- 支持读取 `PORT`、`OPENAI_API_KEY`、`OPENAI_MODEL`、`DATABASE_URL`、`ENCRYPTION_KEY`、`DOCKER_HOST`。
- 默认 `PORT=5000`、`OPENAI_MODEL=gpt-5.2`。

3. 数据库接入
- 建表 `users`、`deployments`。
- 封装基础 CRUD（创建部署、更新状态、查询部署）。

1. 部署接口
- `POST /api/deploy`： Telegram Token + 创建部署。
- `GET /api/deploy/:id`：查询并返回部署状态。

1. 容器部署模块
- 使用 Dockerode 创建并启动 OpenClaw 容器。
- 注入环境变量：`TELEGRAM_TOKEN`、`OPENAI_API_KEY`、`OPENAI_MODEL`。
- 成功与失败分别更新部署状态。

1. Docker 化
- 编写 `backend/Dockerfile`。
- 编写 `backend/docker-compose.yml`，暴露 `5000`。
- 增加健康检查路由 `GET /health`。

1. VPS 部署
- 上传代码或通过 Git 拉取。
- 执行 `docker compose up -d --build`。
- 验证 `http://47.253.91.60:5000/health` 或部署接口可访问。

## 完成标准
- `POST /api/deploy` 返回 `deployment_id`。
- `GET /api/deploy/:id` 能获取 `running/failed`。
- 容器可成功启动并运行。
- 后端通过 Docker 在 VPS 常驻运行。
