# 后端需求文档（MVP）

## 1. 目标
- 提供部署 OpenClaw 的最小后端服务。
- 模型固定 GPT-5.2，仅支持 Telegram。
- 对外提供部署创建与状态查询接口。

## 2. 范围
- 部署接口：创建部署、查询部署结果。
- 与 Docker 引擎交互：创建并启动 OpenClaw 容器。
- 持久化部署记录与错误信息。

## 3. 功能需求
- `POST /api/deploy`：创建部署任务。
- `GET /api/deploy/:id`：查询部署状态。
- 仅接受已登录用户请求（Bearer Token）。

## 4. 数据需求
- 记录 `deployments`：用户、状态、错误信息、加密后的 Telegram Token。
- 记录 `users`：用于关联部署。

## 5. 部署与运行
- 后端部署在 VPS，通过 Docker 方式运行。
- 服务监听 `0.0.0.0:5000`。
- Vercel 前端通过代理访问（浏览器 → Vercel → 后端 IP）。
