# Task 12: Deploy OpenClaw 按钮 - 产品需求文档 (PRD)

## 1. 需求概述

### 1.1 背景
在 SimpleClaw 落地页中，用户完成登录后，需要有一个明确的部署按钮来触发 OpenClaw 实例的部署流程。

### 1.2 目标
- 在登录信息下方添加"Deploy OpenClaw"部署按钮
- 按钮调用后端接口进行部署
- 显示部署状态和相关提示信息

### 1.3 参考
- 设计参考: ![Deploy Button](../../../tasks/image-13.png)
- 页面风格: https://shipany.ai/ (深色主题)

---

## 2. 功能需求

### 2.1 按钮位置
- 位于用户登录信息卡片下方
- 与登录模块在同一个区域内

### 2.2 按钮样式
- **文字**: "Deploy OpenClaw"
- **图标**: 闪电图标 (⚡)
- **样式**: 深色/灰色按钮，圆角设计
- **宽度**: 适应内容或全宽

### 2.3 交互逻辑
1. **初始状态**: 按钮可点击，显示 "Deploy OpenClaw"
2. **点击后**: 调用后端部署接口
3. **部署中**: 显示 loading 状态，按钮禁用
4. **部署成功**: 显示成功状态，更新提示信息
5. **部署失败**: 显示错误信息，按钮恢复可点击


### 2.5 部署前置条件检查
- 用户已登录
- 已选择 Model
- 已选择 Channel (Telegram)
- 已输入 Bot Token

---

## 3. 技术实现

### 3.1 API 接口
```typescript
POST /api/deploy
{
  model: string,
  channel: string,
  telegram_token: string
}
```

### 3.2 响应处理
- 成功: 返回 deployment_id，轮询状态
- 失败: 显示错误信息
